## 2. Categories of Data We May Collect

### 2.1 Account Data

When users create accounts, we may collect:

| Category | Examples |
|----------|----------|
| Identification Information | Display name, artist name, email address |
| Account Credentials | Password hashes (not plaintext passwords) |
| Profile Information | Bio, profile pictures, social media links |
| Security Settings | 2FA enrollment, security preferences |
| Policy Acceptance Records | Timestamps of terms acceptance, versions accepted |

### 2.2 Transaction and Activity Data

As users interact with the platform, we may collect:

| Category | Examples |
|----------|----------|
| Master IPO Data | Records of Master IPOs initiated, NFTs minted |
| Purchase Records | NFT purchases, payment records (not full card numbers) |
| V Studio Participation | Session attendance, voting history, chat participation |
| Content Uploads | Audio files, artwork, metadata |
| Wallet Connections | Connected wallet addresses (public, not private keys) |

**Note:** We do NOT collect or have access to private keys. Users maintain custody of their own wallets.

### 2.3 Technical and Usage Data

We may automatically collect:

| Category | Examples |
|----------|----------|
| Device Information | Browser type, operating system, device identifiers |
| Usage Analytics | Pages visited, features used, time spent |
| Log Data | IP addresses, access times, error logs |
| Cookies and Tracking | Session cookies, analytics cookies |

### 2.4 Communication Data

When users communicate with us, we may collect:

| Category | Examples |
|----------|----------|
| Support Communications | Emails, chat transcripts, support tickets |
| Feedback | Survey responses, feature requests |

## 4. Third-Party Integrations

### 4.1 Blockchain Networks

AudiFi integrates with blockchain networks (e.g., Ethereum). Data recorded on-chain is:

- Public and transparent
- Not controlled by AudiFi
- Immutable or difficult to modify

Users should understand that wallet addresses and transaction data are publicly visible on the blockchain.

### 4.2 Payment Processors

AudiFi may integrate with payment processors (e.g., Stripe). When using fiat payments:

- Payment data is processed by the payment processor
- AudiFi receives limited transaction information
- Full payment card data is not stored by AudiFi

### 4.3 Wallet Providers

Users connect their own wallets (e.g., MetaMask). Wallet providers:

- Maintain their own privacy policies
- Control user wallet data
- Are not controlled by AudiFi

### 4.4 Analytics and Monitoring

AudiFi may use third-party services for:

- Usage analytics
- Error monitoring
- Performance tracking

These services have their own data practices described in their privacy policies.

### 4.5 Cloud Infrastructure

AudiFi uses cloud hosting services to:

- Store data
- Run platform services
- Deliver content

Data may be processed on infrastructure provided by third-party cloud providers.
