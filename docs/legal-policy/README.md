## AudiFi Product Model Context

These documents are drafted with the following AudiFi features in mind:

### Master IPO
- Artists can initiate "Master IPOs" for Master recordings
- NFTs (ERC-721C) represent revenue shares in Masters
- Revenue distribution via smart contracts

### V Studio
- Live interactive sessions between artists and communities
- Voting, chat, and participation features

### Tokens
- Artist Coins (ERC-20)
- Access passes (ERC-1155C)
- Staking and liquidity features

### Royalty Mechanics
- ERC-2981 royalty standard integration
- Mover Advantage allocations (10/5/3/1)

## How to Use These Documents

### For Legal Counsel

1. **Review each document** for legal accuracy and completeness
2. **Identify gaps** based on specific jurisdictional requirements
3. **Revise and finalize** with appropriate legal language
4. **Remove DRAFT labels** only after full approval
5. **Provide guidance** on open questions flagged in documents

### For Product/Engineering Teams

1. **Reference the Policy-to-UX Mapping** for UI implementation
2. **Follow Legal Integration document** for backend requirements
3. **Do NOT implement consent flows** until policies are finalized
4. **Coordinate with legal** before any policy-related deployments

### For Operations/Support Teams

1. **Understand enforcement frameworks** outlined in Content & IP Policy
2. **Prepare processes** for takedown handling
3. **Train on artist rights** using the Rights & Ownership Policy
4. **Wait for legal approval** before using documents externally

## Document Standards

All documents in this folder follow these standards:

### Headers

Every document includes:
```
**DRAFT – FOR REVIEW BY QUALIFIED LEGAL COUNSEL**

*Document Version: X.X*
*Last Updated: [DATE]*
*Operator Entity: [AudiFi Operator Entity]*
```

### Placeholders

Documents use placeholders for items requiring legal decisions:
- `[AudiFi Operator Entity]` - Legal entity name
- `[Governing Jurisdiction]` - Jurisdiction for legal purposes
- `[Address Placeholder]` - Physical address
- `[Email Placeholder]` - Contact email addresses
- `[DATE]` - Document dates

### Notes to Counsel

Documents may include notes such as:
```
*[Note to Counsel: This section requires review for...]*
```

These flag specific items needing legal attention.
